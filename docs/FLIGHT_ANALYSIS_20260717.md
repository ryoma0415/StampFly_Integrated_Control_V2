## 1. 今回の飛行で「検証できたこと」(変更不要と確認された項目)

改善点の前に、データで健全性が確定した項目。ここには工数をかけない。

- **機上ヨー回転補償(`pos_ctrl_yaw_sign`)は飛行実証済み**。便Bでヨー+32°を24.9秒保持中の
  XY RMS は 85.0mm で、ヨー0°時(85.0mm)と完全に同一。符号・回転とも正しい。
  「要地上検証」の注記は解消してよい。
- **400Hz 制御ループ**: TLM_CTRL(+89B/25Hz)追加後も実測周期 2499.6±7µs、最悪 2601µs(+4%)が
  1回のみ。ティック落ちゼロ。
- **EKF・ヨー角制御**: 飛行中の縮退(レートダンピング落ち)ゼロ。ヨー追従 σ2〜3°。
  便Aの t=33〜49s に外因性の磁気異常(NIS最大27)があったが、ゲート機構(棄却→ソフト再捕捉)が
  設計どおり防御し、ヨー変動は 9.4°以内に収まった。
- **通信**: 送信失敗 0、レイテンシ中央 31ms、MoCap フレーム欠落 0。
- **PID テレメトリの整合**: P+I+D=指令角速度が roll/pitch で完全一致。ヨーの ±1rad/s
  クランプ発動を便Bで1回初観測(仕様どおりクランプ前後の差として現れた)。
- **±15% スラストクランプは無実**: duty_mean/Thrust0 は最大 1.118 で上限 1.15 に一度も未到達。
  「クランプ起因のリミットサイクル」仮説はデータで否定された。
- **ToF・高度カルマン**: 外れ値イベント 0(|tof−est|>50mm が 0 行)。センサ系は健全。
- **MoCap ヨーは制御に未使用**(コード確認済み: ファームの `pos_mocap_yaw_rad` は診断用で
  制御からは読まれない)。ログ上のヨーフリップ(±90°の解反転、全行の2.5〜3.1%)は
  **解析時の基準値を汚すだけ**で、飛行そのものへの影響は位置データ検証でも棄却された
  (フリップ行の生位置ジャンプは最大15mm、is_outlier 0%)。

## 2. 改善点の詳細(現象 → 原因 → 対策 → なぜ効くか)

### 2.1 【★1】トリムFF(ホバー推力の一次式)の再校正

- **現象**: 上下速度PIDの積分項が飛行中**常時 +7.3% 分の推力を負担**している
  (duty_mean/Thrust0 の平均が 1.073〜1.075)。その結果、±15% クランプ帯の実効余裕が
  上+7.7% / 下−22% と非対称になり、高度の逸脱が上振れ側(+120〜275mm)に偏っている。
- **原因**: `get_trim_duty()` の一次式 `−0.2448×V + 1.5892` が現機体(重量・ペラ状態)の
  実ホバー推力を過小評価している。
- **対策**: [config.hpp](../firmware_stampfly/src/config.hpp) の `trim_duty_intercept`
  1.5892 → **約1.643**(実測 +0.054duty を吸収)。より正確には現機体で
  hover duty vs 電圧を再フィット。
- **なぜ効くか**: FF が実ホバー点に一致すると、①クランプ帯が上下対称の±15%に戻り
  上方向の外乱応答が回復、②積分の定常負担が消えて過渡時の応答が素直になる。
- **検証指標**: duty_mean/Thrust0 の平均が 1.0±0.02 になること。

### 2.2 【★1】電圧→Thrust0 の正帰還の遮断

- **現象**: 上下速度に **0.42〜0.47Hz の指令に無いボビング**(z_dot_ref の同帯域成分の
  約5倍の振幅)。duty とのコヒーレンス 0.90〜1.00 = 閉ループ起因。
- **原因**: Thrust0 が INA3221 の 20Hz 生電圧を直接参照しているため、負荷電流による
  電圧サグ(dV/d(duty)≈−0.7V/duty)経由で **dThrust0/d(duty)=+0.17〜0.19 の正帰還**が
  形成され、ちょうどボビング帯の減衰を削っている(Thrust0 変動は制御出力の約25%相当)。
- **対策**: `get_trim_duty()` に渡す電圧に**1次LPF(時定数1〜2s)**を追加。
  低電圧フェイルセーフ判定は生値のままでよい。
- **なぜ効くか**: ホバーFFは本来ゆっくり変わる量(電池残量)だけを補償すべきで、
  秒スケールのサグ変動を遮断すれば正帰還ループが切れ、ボビングの減衰が戻る。
- **検証指標**: alt_velocity の 0.3〜1.0Hz 帯 RMS(現状 88〜99mm/s)→ 60mm/s 以下。

### 2.3 【★1】ヨーレートクランプの縮小(duty 天井対策)

- **現象**: モータ duty=0.95 天井への到達が便Aで 2.95%(電圧 3.42V の終盤区間では
  **12.8%**)。天井到達後 1.5 秒以内の XY 誤差は非到達時比 **+37%**。便Bでは天井→高度
  +24〜34cm 浮上→(高度0.5m超で)トラッキング喪失→XY 253mm という最悪連鎖の起点。
- **原因**: ①ホバー duty 中央値 0.84 と高く上限 0.95 までの余裕が 0.11 しかない、
  ②**ヨー角速度ループ出力がミキサ差動の最大消費者**(|出力|×0.25 > 0.1duty が時間の7割、
  天井到達との相関 +0.76〜0.79)、③ヨーIが常時 −0.32〜−0.37duty の反トルク補償を
  している(機体の非対称)。
- **対策**:
  1. `yaw_rate_limit_rad_s` 1.0 → **0.4〜0.5**(実運用のヨーレート指令 RMS は 0.15rad/s、
     便Bの回頭ランプでも 1.03 が1回だけなので通常運用への影響は僅少)
  2. ヨーIの定常負担の原因点検(プロペラ摩耗・取付。左右で +0.02〜0.025duty の差)
  3. (中期)差動優先の duty 配分ミキサ(飽和時に集合推力側を削って姿勢差動を保護)
- **なぜ効くか**: 天井到達はヨートルク要求が引き金になっている頻度が高く、上限を
  実運用レンジに絞るだけで飽和イベント(=高度・XY 双方の外乱源)が減る。
- **検証指標**: duty 0.95 到達行 0%(満充電〜3.45V の区間で)。

### 2.4 【★2】MoCap 環境の改善(マーカー配置とカメラ範囲)

- **現象**: 便Bで**トラッキング完全喪失が2回**(rb_marker_count=0、各≈0.5秒、
  confidence 0.05)。いずれも高度オーバーシュート中の **alt≈0.49〜0.51m** で発生し、
  XY 閉ループが開ループ化(cmd_xy_valid=0 送信)→ 復帰後の再シードで XY 誤差 253mm まで
  拡大(便Bの最大逸脱の直接原因)。また RB のヨー解が±90°反転する現象が両便で頻発
  (72〜184回/飛行)しており、解の一意性が弱い。
- **原因**: ①キャプチャボリュームの有効上限が高度約0.5m 付近にある(便Aは最大0.48mで
  喪失ゼロ、と整合)、②マーカー配置が90°回転にほぼ対称で、Motive のリジッドボディ解が
  縮退している(ヨー解反転はその症状。制御には無害だが追跡ロバスト性の低さを示す)。
- **対策**: ①カメラの向き/キャリブレーションを見直し、高度オーバーシュートが到達する
  **0.65m 程度まで**カバーする。②マーカー1個を非対称位置に移設/追加して 90°対称性を崩す。
- **なぜ効くか**: 喪失そのものが起きなければ「高度が動くと XY もずれる」連鎖の最終段
  (開ループ化)が切れる。解の一意性が上がれば低視認状態でのロック維持も改善する。
- **検証指標**: 飛行中の tracking_valid=0 が 0 回、mocap ヨーの ±90° ステップが激減。

### 2.5 【★2】高度内側ループ(z_dot_pid)の強化

- **現象**: 上下速度の追従が弱い(alt_velocity の変動 std が z_dot_ref の 3.2〜3.6倍、
  追従誤差 RMS は指令 RMS の約4倍)。支配周波数での位相遅れ −94〜−103°。
- **原因**: z_dot_pid kp=0.08 が小さく、カルマン速度の約94ms 遅れ+25Hz 離散化と
  合わさって減衰不足。
- **対策**: kp 0.08 → **0.10 → 0.12** と1段ずつ(Ti 0.95→0.8 も可、Td 0.08 維持)。
  §2.1・§2.2 の後に実施(FFずれと正帰還が残ったままゲインを上げると逆効果)。
- **なぜ効くか**: 内側ループの帯域・減衰が上がると、外側 alt_pid が生成する
  Z_dot_ref に実速度が追従できるようになり、ボビングとワンダリングの両方の土台が締まる。
- **検証指標**: §2.2 と同じ+高度誤差 RMS 80mm → 50mm 以下。

### 2.6 【★3】高度外側ループ(alt_pid)の強化 — 内側の後で

- **現象**: 高度誤差パワーの8割超が 0.3Hz 未満の**準周期ワンダリング(0.08〜0.11Hz、
  RMS 80mm)**。明確なリミットサイクルではない(自己相関第1ピーク r=0.13〜0.16)。
- **原因**: alt_pid kp=0.38 の実効帯域(≈0.06Hz)が低く、ボビング等で生じた高度ドリフトの
  引き戻しに8〜10秒かかる。Z_dot_ref の振幅も最大0.15m/s と小さい。
- **対策**: **内側(§2.5)を強化した後に** kp 0.38 → 0.5〜0.55。Td 0.5・Ti 10 は維持
  (定常誤差は+17〜19mmと小さい)。
- **なぜ効くか**: 引き戻しが速くなりワンダリング振幅が縮む。順序を守るのは、内側が
  遅いまま外側を上げると 0.1Hz 帯がむしろ悪化するため。

### 2.7 【★3】XY 位置ループのゲイン向上

- **現象**: XY 誤差はイベントを除いても**周期9〜13秒の低周波ふらつきが支配的**
  (RMS 95mm、>0.1m が26〜32%)。高度振動(6.4〜8s)とは帯域が分離しており独立現象。
- **原因**: 位置PID(kp 0.139 / kd 0.204)のループゲイン・減衰不足。出力の±5°飽和は
  1%未満、送信誤差クランプ発動 0% で、**ゲインを上げる余地が明確にある**。
- **対策**: config.hpp の `pos_kp` 0.139 → **0.17 前後**、`pos_kd` 0.204 → **0.25 前後**を
  1段ずつ(±25%程度)。§2.3 の duty 天井対策後に実施(飽和中のゲイン増は逆効果)。
- **なぜ効くか**: ふらつきの主因が帯域不足なので、比例・微分の同時増強で位相余裕を
  保ったまま剛性を上げられる。
- **検証指標**: XY RMS 95mm と >0.1m 率 26〜32% の低減(1飛行ごとに比較)。

### 2.8 【★4】その他(個別イベントの再発防止・中長期)

1. **喪失復帰のソフト化**(pc_server): トラッキング喪失→再シード後の位置ジャンプが
   機上XYの D 項(kd 0.204)を蹴り、復帰直後に誤差が 160→253mm へ再拡大した。
   `cmd_xy_valid` 復帰後 0.3〜0.5 秒は送信誤差を 0→100% にランプインする。
2. **EKF 磁気バイアス |b_m| の成長対策**: 飛行後半に単調増加し便Aで 18.6µT
   (凍結閾値20µTの93%)に到達。2分超の飛行や消耗バッテリで飛行中凍結→ヨー縮退が
   現実化しうる。当面は1バッテリ90秒以内の運用+UI に |b_m|>15µT 警告。中期的には
   ffcal への電圧/duty 依存項の追加を検討。
3. **Z_REJECT ゲートの見直し**: z残差>12µT ゲートが飛行中 91〜95% 常時成立しており
   判別力ゼロ。FF補正に z 成分を追加するか、FF有効中は閾値を duty 連動で緩める。
4. **機上高度推定の +4〜5cm バイアス**: est−MoCap 実測差が静的に +43〜49mm あり、
   実ホバー高度は目標より約3cm低い。推定器補正か alt_ref オフセットで解消可能(優先度低)。
5. **pitch 角速度Dの 9.2Hz 成分**: D出力に高周波が乗っている(rms 0.18〜0.20duty)。
   角速度D項に 4〜5Hz LPF を検討。ただし 25Hz テレメトリではエイリアスの可能性があるため、
   本対応の前に機上 400Hz での真の周波数確認を推奨。
6. **離陸オーバーシュート(+125〜167mm)**: 2段目標(0.25→0.3m)か、ランプ終了時の
   alt_pid 積分初期化(優先度低)。
7. **(解析品質のみ)MoCap ヨーフリップの検疫**: 制御には無関係と確定したが、
   ログ上の「MoCap 真値」を汚しヨー推定精度の評価値を膨らませる。ヨー評価を行う際は
   フレーム間ジャンプ>30°の行を除外して統計を取ること(PC側実装は低優先)。
