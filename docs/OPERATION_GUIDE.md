# StampFly Integrated Control — 段階的安全飛行手順

本書はオペレータ向けの飛行運用手順。セットアップと UI の詳細は
[../README.md](../README.md)、フェイルセーフの正典は
[PROTOCOL.md](PROTOCOL.md)「タイミング・フェイルセーフ(規範)」。

**原則: 各フェーズの確認項目がすべて合格するまで、次のフェーズへ進まない。**
リスクの低い順(机上 → 機体電源 OFF での通信確認 → 地上確認 → 短時間ホバー →
本飛行)に進める。

---

## 1. 飛行前チェックリスト

- [ ] 2m×2m 以上の空間を確保(Position モードは MoCap 計測範囲内)
- [ ] 周囲に人・ペットがいない。壊れやすい物を片付けた
- [ ] バッテリー充電済み(UI の電圧表示で 3.5V 以上が目安。3.4V 未満は飛行不可)
- [ ] プロペラの損傷なし。プロペラガード装着(推奨)
- [ ] 機体は平らな床の中央に水平に置く
- [ ] PC のキーボードに手が届く位置にいる(**SPACE = 緊急停止**)
- [ ] 飛行は **release ビルド**のファームで行う(README §3)

## 2. 緊急時の操作(最初に覚える)

下から順にエスカレーションする。**迷ったらまず SPACE**。

| レベル | 操作 | 効果 |
|---|---|---|
| 1 | UI の **STOP** ボタン(どちらのタブにもある) | CMD_STOP 送信 → 即時着陸。**全飛行状態で受理**(TAKEOFF/LANDING 中含む) |
| 2 | **SPACE キー** | どこからでも(フォーカス位置に関わらず)即 STOP。連打してよい |
| 3 | **機体の電源スイッチ OFF** | 最終手段。その高度から落下するため、低高度で機体に安全に近づける場合のみ |

補足(知っておくと落ち着いて対処できる):

- STOP 送信後 600ms 以内に着陸イベントがなければ、pc_server が**自動で再送**する
  (最大3回)。「応答がありません」警告が出ても機体側フェイルセーフが控えている。
- **PC・サーバ・リレーのどれが落ちても機体は自分で着陸する**: 飛行中に
  CMD_SETPOINT(50Hz ハートビート)が 200ms 途絶えると水平化、500ms で
  自律着陸(link_loss)に入る(§9 の表)。USB を抜く・サーバを強制終了する
  だけでも結果的に着陸する。
- OverG(衝突など >2.0g)を検出するとモータ即停止 → COMPLETE になる。復帰は
  機体を回収・静置してから UI の **Re-arm (RESET)**(推定高度 0.15m 未満でのみ受理)。

## 3. フェーズ0: 机上検証(リスク: なし、ハード不要)

```sh
# プロトコル実装の一致テスト
cd protocol && python3 -m pytest tests/ -q          # 全件 pass

# サーバのロジックテスト(フェイク serial/NatNet)
cd pc_server && .venv/bin/python -m pytest tests/ -q # 全件 pass

# ファームのコンパイル確認
~/.platformio/penv/bin/pio run -d firmware_stampfly -e release
~/.platformio/penv/bin/pio run -d firmware_relay -e release
```

すべて成功すること。venv が無ければ README §4.1 を先に実施する。

## 4. フェーズ1: 通信確認 — 機体電源 OFF(リスク: なし)

**機体の電源は OFF のまま**、PC⇔リレー区間だけを確認する。

1. リレーを USB で PC に接続し、pc_server を起動(README §5)、UI を開く。
2. ヘッダで機体プロファイルを選択し、リレーのポートを選んで「接続」。

確認項目:

- [ ] **シリアル**インジケータが緑
- [ ] **リレー**インジケータが緑(機体が OFF でも RLY_SET_TARGET の ACK は返る。
      黄のままなら README §7 のリレー欄を確認)
- [ ] コンソールに「リレーターゲット設定完了: <MAC> ch<N>」が出る
- [ ] **機体**インジケータは消灯のまま(電源 OFF なので正常)
- [ ] リレー統計 `relay ↑.. ↓.. err:0` が毎秒更新される(err が増えないこと)
- [ ] **STOP ボタンと SPACE キーを実際に押す**: コンソールに「CMD_STOP 送信」、
      しばらくして「CMD_STOP への応答がありません(再送上限到達)」警告
      —— 機体 OFF ではこれが**期待どおりの動作**。緊急停止の操作を体に覚えさせる

## 5. フェーズ2: 機体地上確認 — プロペラ回転なし(リスク: 低)

1. 機体を平らな床に置き、電源 ON。INIT → CALIBRATION → WAIT と進む間
   (数秒+静定 3 秒)は**機体を動かさない**(ジャイロ較正と AHRS リセット)。
2. UI で確認:

- [ ] **機体**インジケータが緑、状態バッジが **WAIT — 待機**
- [ ] 電圧が表示され 3.5V 以上(黄/赤なら充電してから)
- [ ] レイテンシが数値表示される(セットポイント往復が機能している証拠)
- [ ] 機体を**手で**ゆっくり傾けると姿勢バー(Roll/Pitch)が追従する
- [ ] 高度スライダを動かすと「高度(現在 vs 目標)」の目標マーカーが追従する
      (高度は離陸前から変更可能。これが離陸目標高度になる)

## 6. フェーズ3: 短時間ホバー(リスク: 中)

§1 のチェックリストを完了してから。**Posture タブ**で行う。

1. 高度スライダを 0.3m 程度に設定。Roll/Pitch はスライダ中央(0°)のまま。
2. ログ保存トグルを ON(後の較正・解析のため推奨)。
3. **START** → 確認ダイアログで OK → **1〜2 秒で STOP**(または SPACE)。

- [ ] まっすぐ離陸し、STOP で降下して着陸する
- [ ] **流れた方向(前後左右)をメモする**(§7 の較正に使う)
- [ ] 異常(片側に強く倒れる・異音)があれば即 SPACE → 電源 OFF し、
      プロペラ・モータを点検

問題なければ **5〜10 秒の通常ホバー**を行い、安定性とドリフト方向を再確認する。
複数回の離着陸(START→STOP→START→STOP)も正常に動くこと。

## 7. 機体プロファイル較正(角度バイアス)

機体には個体差があり、指令 0° でも一方向に流れる(ドリフトする)ことがある。
このドリフトを **PC 側プロファイルのバイアス**(`pc_server/config/airframes.json`
の `roll_bias_deg` / `pitch_bias_deg`)で打ち消す。ファームには入れない。

**考え方(ドリフト方向からの逆算)**: ホバー中に流れる方向 = 機体が実際に
傾いている方向。これを打ち消すには、**流れる方向と逆向きに傾ける指令**
(=バイアス)を常時加算する。必要量は流れの速さに応じて 0.5°〜2° 程度が目安。

**手順(Posture モードのスライダで「測る」)**:

1. バイアス 0 のプロファイルで 0.3m ホバーし、ドリフト方向と速さを観察する
   (5 秒程度。ログ ON 推奨)。
2. ホバーを続けたまま、**流れと逆方向に Roll/Pitch スライダを 0.5° 刻みで動かし、
   ドリフトが止まる値を探す**。符号に自信がなければ小さく振ってみて、
   悪化したら逆へ。バッテリー残量に注意し、必要なら着陸して再開する。
3. ドリフトが止まったときの**スライダ値がそのままバイアス値**。値を読み取って
   着陸(STOP)する。
4. UI ヘッダの**「編集」**で機体プロファイル編集を開き、該当プロファイルの
   `roll_bias_deg` / `pitch_bias_deg` にスライダ値を記入して**保存**
   (例: `-1.0` / `0.5`)。`notes` に測定日を残す。サーバの再起動は不要 ——
   保存と同時に選択中プロファイルのバイアスが再適用される(飛行中は
   選択中プロファイルの変更が拒否されるため、**必ず着陸後に保存**する)。
5. スライダ中央(0°)で再ホバーし、その場に留まることを確認。残るようなら
   1°→0.5°→0.25° と刻みを細かくして繰り返す。

検証・微調整のヒント:

- ログの `roll_ref_deg`/`pitch_ref_deg` は**バイアス加算後**の送信値、
  `roll_bias_deg`/`pitch_bias_deg` は適用中バイアス(LOG_STRUCTURE.md)。
  引き算すればスライダ操作分が復元できる。
- Position モード運用後の上級者向け確認: 安定ホバー中のログの `pid_x_i`/`pid_y_i`
  (XY PID の I 項)が大きな定常値を持つ場合、それは XY PID がバイアス相当を
  肩代わりしている兆候。その分をプロファイルへ移すと I 項の負担が減る
  (符号は座標変換を挟むため、移した後に必ず実飛行で確認する)。

## 8. フェーズ4: 本飛行

### 8.1 Posture モード(手動の姿勢・高度指令)

- Roll/Pitch スライダは飛行中のみ操作可。**小さく動かして必ず中央に戻す**
  (「中央に戻す」ボタン)。サーバ側でスルーレート 30°/s に制限されるが、
  指令を出しっぱなしにすれば機体は流れ続ける。
- 高度スライダは 0.1–1.0m。変更は 0.3m/s に制限されて反映される。
- 着陸は STOP。最大飛行時間 120 秒で自動着陸する(§9)。

### 8.2 Position モード(MoCap による位置保持)

前提: Motive/NatNet 設定済み(README §4.3)。

1. **離陸前に** Position タブで確認:
   - [ ] MoCap インジケータが「受信中」(緑)
   - [ ] 座標表示と XY プロットが、機体を手で動かすと追従する
   - [ ] 値が制御座標系として妥当(原点・軸方向が想定どおり)
2. 目標を「**この場で**」プリセットにする(現在位置 XY を目標化。
   離陸直後の横移動を防ぐ)。目標 Z は 0.3m 程度から。
3. **START**(MoCap が新鮮でないと拒否される)。
4. 安定したら目標 X/Y を **0.1〜0.2m 刻み**で動かして追従を確認。
   「原点」プリセットで (0,0) へ戻せる。
5. 着陸は STOP。MoCap が 300ms 途絶えると水平固定+警告、2 秒で自動着陸する(§9)。

### 8.3 終了手順

1. STOP → 着陸と WAIT 復帰を確認
2. ログ保存トグル OFF(ファイルが閉じられる)
3. UI で「切断」→ 機体の電源 OFF → バッテリーを外して保管

## 9. フェイルセーフ一覧(PROTOCOL.md より転記)

正典は [PROTOCOL.md](PROTOCOL.md)。PC 側閾値の実効値は
`pc_server/config/server.json`(`failsafe` 節)、ファーム側は
`firmware_stampfly/src/config.hpp`。

| 条件 | 動作 | 実装場所 |
|---|---|---|
| CMD_SETPOINT 途絶 >200ms(飛行中) | roll/pitch を水平(バイアス込み0)へ、alt_ref 維持 | ファーム |
| CMD_SETPOINT 途絶 >500ms(飛行中) | LANDING へ遷移(reason=8 link_loss) | ファーム |
| 低電圧(<3.34V 100tick連続) | 飛行中: LANDING(reason=4)。WAIT: START拒否(reason=5) | ファーム |
| OverG(>2.0g) | モータ即停止 → COMPLETE(reason=7)。CMD_RESETでのみ復帰 | ファーム |
| 最大飛行時間 120s | LANDING(reason=3) | ファーム |
| MoCap 途絶 >300ms(Positionモード) | setpoint を水平に固定+UI警告 | pc_server |
| MoCap 途絶 >2s(Positionモード) | CMD_STOP 送信(自動着陸) | pc_server |
| STOP 送信後 600ms 以内に LANDING/WAIT イベントなし | CMD_STOP 再送(最大3回)+UI警告 | pc_server |
| シリアル切断 | UI赤色警告(機体側は上記リンク喪失で自律着陸) | pc_server |

このほか UI/サーバ/ファームの三層で指令クランプがかかる
(UI ±5°・0.1–1.0m / サーバ ±10°+30°/s・0.1–1.2m+0.3m/s / ファーム ±30°・0.05–1.5m。
ARCHITECTURE.md「安全クランプ(多層)」)。

## 10. よくある問題(飛行時)

| 症状 | 対処 |
|---|---|
| START しても離陸しない | 状態バッジを確認。WAIT 以外では受理されない。「離陸拒否: START_REJECTED_…」がコンソールに出ていれば理由(低電圧/未準備)を解消 |
| ホバーが一方向に流れる | §7 のバイアス較正。較正済みなら床の傾き・プロペラ損傷・気流を疑う |
| 飛行中に勝手に着陸した | コンソールのイベント(reason)を確認: link_loss=電波/USB、low_voltage=充電、max_flight_time=仕様(120s) |
| COMPLETE から動かない | OverG 後。機体を回収・静置(高度 0.15m 未満)して Re-arm (RESET) |
| Position で警告「MoCap 途絶」 | マーカーの隠れ・計測範囲外。頻発するなら Motive のカメラ配置と `control.json` のフィルタ設定を見直す |
