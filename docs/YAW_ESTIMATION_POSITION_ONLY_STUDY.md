## 0. 背景と目的

### 0.1 なぜ「位置のみ」に限定するか

MoCap(OptiTrack)は剛体の 6DOF を持ち、ヨー角も直接与えられる。これを EKF のスカラー観測に
入れれば現行のヨードリフトは無条件に根治する(前回検討の「案1」)。しかし **MoCap でヨーを
取得できるのは屋内のモーションキャプチャ環境という特別なケースに限られ、汎用性がない**。

一方、**位置情報は GPS で代替可能**である。したがって本検討では外部入力を
**「位置情報のみ」** に限定し、

> GPS(屋外) / MoCap(屋内・開発時) / フロー積分(位置源が無いとき) といった
> **位置源を差し替えるだけで、同一のヨー推定器が動く** 構成

を目標とする。このとき「MoCap ヨー」は使わないが、**「MoCap 位置」は GPS 位置の高精度な
代替として開発・検証に使える**(避けるのは MoCap のヨーだけ)。

### 0.2 出発点となった記載(EKF-GSF)

PX4 / ArduPilot は磁気センサを使わず GPS+IMU からヨーを出す **EKF-GSF(Gaussian Sum Filter)**
を実装している。要点は次の通り(引用元の主張):

- 真のヘディングは、GPS 等の絶対位置・速度を融合しつつ **機体が(線形)加速しているときに
  可観測**になる。加速度計は機体座標の比力を測り、GPS 速度の微分は世界座標(NED)の加速度を
  与える。同一の加速度ベクトルの 2 表現を結ぶ重力軸まわりの回転 = ヨー が解ける。
- **ホバリング中・等速飛行中は水平加速度がゼロ → 情報ゼロ → ヨーはジャイロバイアスと共に
  ドリフト**する。
- 実装は NE 速度とヨー角を状態とする複数の 3 状態 EKF の束(ガウス和)。単一 EKF だと強い
  非線形・多峰性で局所解にハマるため。
- コンパス搭載機でも GPS 由来ヨーで常時クロスチェックし、壊れたらリセットする(ride-along)。

本書はこの記載が StampFly に当てはまるかを検証し、**フローを持つ StampFly では GSF の
「加速必須」制約を緩められる**ことを示す。

## 1. 結論(要約)

1. **位置/速度からヨーを推定するのは理論的に可能**。ただし「位置がヨーを直接観測する」のでは
   ない。位置は瞬時にはヨーを一切拘束しない。ヨーは **同一物理ベクトルの機体座標表現と
   世界座標表現を結ぶ回転** として現れる。
2. その経路は 2 つある:
   - **機構 A(速度整合、フロー必要)**: 機体速度 v^B(フロー)と世界速度 v^W(位置微分)を
     結ぶ。**‖v‖ > 0 なら可観測(等速直進でも可)**。
   - **機構 B(加速度整合、GPS+IMU のみ = GSF 本体)**: 機体加速度 a^B(加速度計)と
     世界加速度 a^W(速度微分)を結ぶ。**‖a_水平‖ > 0、線形加速時のみ可観測**。
3. **機構 A は機構 B より可観測エンベロープが厳密に広い**(A は等速並進でも効く、B は加速中のみ)。
   PX4/ArduPilot が「GSF は加速が必要」「Plane で有効・Copter で弱い」と言うのはフロー無しで
   機構 B だけだから。**フローを持つ StampFly はこの不利を部分的に覆せる**。
4. ユーザーの問い「位置/フローを KF に含めてヨー精度を上げられるか」への直接回答は **Yes。
   ただし現行 4 状態(ψ, b_g, b_mx, b_my)に位置観測を足すだけでは不可**。位置とヨーの結合は
   必ず速度を経由するため、**状態ベクトルに水平速度を入れる真のフィルタ拡張**が要る。
5. **磁気センサは捨てるべきではない**。磁気(ホバでも効く)と機構 A/B(並進・加速時に効く)は
   担当領域が理想的に相補。ホバ主体の StampFly では磁気がホバヨー保持の主役になる。
6. **GSF のマルチ仮説(ガウス和)は StampFly には当面不要**。磁気で初期ヨーが ±20〜30° 内に
   入るため単一の拡張 EKF で多峰性の罠を回避できる。GSF はメインヨー破綻検出の ride-along
   クロスチェックとしてのみ、完全磁力レス化を決断した時にフル実装する。

## 2. 可観測性の理論

### 2.1 なぜ位置は瞬時にはヨーを拘束しないか

ヨー角 ψ は機体の向きであり、機体の重心「位置」そのものには現れない。位置観測 z = p^W + n の
ヤコビアン ∂z/∂ψ = 0 である(位置は並進自由度、ヨーは回転自由度)。したがって位置を EKF に
入れても、それが **速度・加速度を通じて姿勢と結合していなければ** ヨーには一切効かない。

結合は運動学から入る。世界座標速度 v^W と機体座標比力 a^B の関係:

```
v̇^W = R(q)·a^B + g^W          (R(q) = 機体→世界の回転行列, g^W = [0,0,g])
```

水平成分だけを取り出し、Roll/Pitch(チルト)を既知として重力とチルト寄与を分離すると、
残る回転自由度は **重力軸まわりの回転 = ヨー R_z(ψ) だけ** になる。ここにヨーが現れる。
以下、水平 2 成分に注目し R_z(ψ) を 2×2 回転行列として扱う。

### 2.2 機構 A: 速度整合(フロー × 位置)

状態を最小に x = [ψ, v^W](v^W ∈ R²: 世界座標水平速度)とする。2 つの観測:

```
z_pos  = p^W + n_p                        (位置。時間微分で v^W を拘束)
z_flow = R_z(ψ)ᵀ · v^W + n_f              (フロー: 機体座標水平速度)
```

フロー実測式(FRD 系、`OPTICAL_FLOW_STUDY.md` で実機検証済み):

```
vx_body = (flow_x − q)·d,   vy_body = (flow_y + p)·d
   p,q = ロール/ピッチ角速度, d = 対地距離(ToF)
```

**構造の核心**: 位置観測は v^W を「世界座標系で」固定し、フロー観測は同じベクトルを
「機体座標系で」測る。同一物理ベクトルの 2 座標系表現を結ぶ回転がまさに ψ である。
2 次元回転は非零ベクトル 1 本の対応から一意に決まる:

```
ψ = atan2(v^W_y, v^W_x) − atan2(v^B_y, v^B_x)    (‖v‖ ≠ 0 のとき)
```

EKF のヤコビアン:

```
∂z_flow/∂ψ = d(R_z(ψ)ᵀ)/dψ · v^W = R_z(ψ)ᵀ·Sᵀ·v^W,   S = [[0,−1],[1,0]]
⇒ ‖∂z_flow/∂ψ‖ = ‖v^W‖
```

**ヨー方向の観測感度は機体速度の大きさそのものに等しい。** 可観測性グラミアンの ψ 成分は
窓 [0,T] で

```
G_ψψ = Σ_k ‖v(t_k)‖² / σ_f²   ⇒   σ_ψ ≳ σ_f / sqrt(Σ_k ‖v(t_k)‖²)
```

定量評価(σ_f ≈ 0.1 m/s、SQUAL 良好時):

| 運動 | ‖v‖ | 1 サンプル σ_ψ | 2 秒窓(50Hz, 100 サンプル) |
|---|---|---|---|
| 定常ホバ | 0.05–0.1 m/s | 45–90°(実質不可観測) | 収束せず |
| 緩い並進 | 0.3 m/s | — | 約 2° |
| 並進 | 0.5 m/s | — | 約 1.1° |

**軽い並進(半径 20–30cm の緩い円・8 の字)を混ぜるだけで、磁気と完全独立に ψ を数°精度で
拘束できる。** 縮退するのは ‖v‖ ≈ 0(定常ホバ)のみ。

### 2.3 機構 B: 加速度整合(GPS+IMU のみ = GSF 本体)

フローが無くても、加速度計と GPS だけでヨーが立つ経路がある。状態 x = [ψ, v^W]:

```
z_acc = R_z(ψ)ᵀ · a^W_水平 + n_a      (加速度計: 機体座標比力の水平成分, 重力・チルト除去後)
a^W_水平 = d(v^W)/dt ≈ (GPS 速度の差分)   (世界座標水平加速度)
```

構造は機構 A と同型で、「速度」を「加速度」に置き換えただけ:

```
∂z_acc/∂ψ = R_z(ψ)ᵀ·Sᵀ·a^W_水平
⇒ ‖∂z_acc/∂ψ‖ = ‖a^W_水平‖
```

**ヨー感度は水平加速度の大きさに等しい。** よって:

- **ホバ(a ≈ 0)**: 不可観測。
- **等速直進(a ≈ 0)**: **不可観測**(← 機構 A との決定的な違い)。
- **加速・減速・旋回(a ≠ 0)**: 可観測。

### 2.4 機構 A と B の非対称性(StampFly の勝ち筋)

| | 機構 A(フロー+位置) | 機構 B(GPS+IMU) |
|---|---|---|
| 必要な励振 | 並進(‖v‖ > 0) | 加速(‖a‖ > 0) |
| 等速直進 | **可観測** | 不可観測 |
| ホバ | 不可観測 | 不可観測 |
| 磁気外乱 | 不感 | 不感 |
| 追加センサ | フロー | 不要 |

**機構 A の可観測エンベロープは機構 B を厳密に包含する。** PX4/ArduPilot が
「GSF は Plane で自動・Copter で明示設定が必要」「良好な GPS 速度に依存」と言うのは、
飛行機は常時前進(A/B 常時アクティブ)だが、マルチコプターはホバできてしまい、かつ彼らは
フロー無しで機構 B しか持たないから。**フローを積む StampFly は機構 A を持てるため、
Copter 側の不利を部分的に覆せる**(加速していなくても緩い並進でヨーが立つ)。

### 2.5 ジャイロバイアスとの結合 — ドリフトの真因

ヨーは ψ̇ = gyro_z − b_gz で伝播する。b_gz(ヨージャイロバイアス)は **ψ が可観測なときにだけ
同時に可観測**になる。したがって:

```
ホバ・等速 → 機構 A/B 死ぬ → ψ も b_gz も不可観測 → 両方が一緒にドリフト
```

これは現行 4 状態ヨー EKF のホバ中 ψ/b_m 縮退(`FLIGHT_ANALYSIS_20260717.md`)と
**まったく同じ構造**である(磁気バイアス b_m の代わりに b_gz、磁気観測の代わりに位置/速度観測)。

∴ 正しいアーキテクチャは:

> **並進・加速の窓で ψ と b_gz を「両方」しっかり pin し、ホバ中はよく推定された b_gz で
> ジャイロを積分してコーストする。**

ホバ中のヨー保持品質は、直前の可観測窓で b_gz をどれだけ正確に推定できたかで決まる。
現行 4 状態 EKF が既に b_g を状態に持っているのは、この意味で正しい。

## 3. 統合フィルタの設計

### 3.1 なぜ「4 状態に位置を足すだけ」では不可か

§2.1 の通り位置のヨーヤコビアンは 0 なので、位置観測を現行 4 状態(ψ, b_g, b_mx, b_my)に
足してもヤコビアン行が [0,0,0,0] になり、ヨーに情報が入らない。結合は速度経由でしか起きない。
**状態に水平速度を入れる = 本物のフィルタ拡張が必須**(前回検討の「案1: スカラー観測追加」
とは質が違う)。

### 3.2 最小統合フィルタ(水平-ヨー EKF)

```
状態: x = [ψ, b_gz, vN, vE, (bax, bay)]        (5〜7 状態)

予測:
    ψ̇   = gyro_z − b_gz
    v̇^W = R_z(ψ)·(a^B − b_a) + g               ← ここで ψ が速度予測に入る = 機構 B が暗黙に内包

観測:
    位置(GPS/MoCap)  → h = p^W = ∫v^W          (速度を時間方向に拘束)
    フロー           → h = R_z(ψ)ᵀ·v^W          (機構 A を明示注入)
    磁気(FF 補正付き) → h = R_z(ψ−ψ0)·B0h + b_m  (ホバ担当。現行 EKF の観測をそのまま)
```

**設計の要点**:
- 機構 B は **予測ステップ**(機体加速度を R_z(ψ) で世界へ回す部分)から自動的に出てくる。
  明示的な「加速度観測」を書かなくても、GPS 位置/速度観測が予測を拘束する過程でヨーに効く。
- 機構 A は **フロー観測**として明示的に入れる。
- これは実質 `stampfly_ecosystem` の 15 状態 ESKF の「水平-ヨー部分集合」であり、
  前回検討の案2(フロー速度 KF)と案3(速度ペア観測)を統合・格上げした形。

### 3.3 磁気センサの位置づけ(相補的)

| センサ | 効く条件 | 弱点 |
|---|---|---|
| 磁気(FF 補正付き) | **ホバでも効く**、低レート絶対ヨー | モーター電流で汚染(高スロットル過渡で最悪) |
| 機構 A(フロー+位置) | 任意の並進 | ホバで死ぬ、磁気外乱には不感 |
| 機構 B(GPS+IMU 加速) | 線形加速時 | ホバ・等速で死ぬ、磁気外乱には不感 |

担当領域がきれいに噛み合う: **磁気がホバ(A/B が死ぬ領域)を、A/B が高スロットル並進過渡
(磁気が最も汚染される領域)を担当**する。だから磁気は「レガシー」ではなく、ホバ主体の
StampFly ではむしろホバヨー保持の主役として残す価値がある。ArduPilot/PX4 がコンパス搭載機
でも GPS 由来ヨーで常時クロスチェックするのは、この相補関係を安全側に使っているということ。

### 3.4 GSF のマルチ仮説は必要か

ψ は R_z(ψ) の中で周期的 = **多峰性**。初期ヨーが 180° ずれると単一 EKF は誤った点で線形化し
誤収束する。GSF は 5 個の 3 状態 EKF [vN, vE, ψ] を等間隔(0/72/144/216/288°)にシードして
尤度加重で大域収束を解く = **初期化問題の解**。

StampFly の判断:
- **磁力計がある限りフル GSF は不要**。FF 補正付き磁気が最悪でも初期ヨーを ±20〜30° 内に
  置くので、単一の拡張 EKF を磁気ヨー近傍で線形化すれば 180° ずれの罠にはまらない。
- **完全磁力レス化する場合のみ** GSF か初期化マヌーバ(引用文の「アーム前に円を描いて歩く」=
  StampFly なら飛行開始時の小さな指令並進)が必須になる。
- 磁気搭載のまま GSF を **ride-along(並走ログのみ)** で走らせ、メインヨーが GPS 由来ヨーから
  乖離したら破綻検出→リセットを促す使い方は、安全側の価値が高い(引用文の「業界の本音」)。

## 4. StampFly 固有の制約(正直な整理)

1. **屋内は GPS 不可**。位置源は屋内=MoCap(開発時)/フロー積分(ドリフトあり)、屋外=GPS。
   **ヨー推定器のコードは同一で、位置源だけ差し替える** — これが望む汎用化そのもの。
2. **MoCap 位置 = GPS 位置の高精度代替**として開発に使える(避けるのは MoCap ヨーのみ)。
   拡張ヨーフィルタを MoCap アリーナで開発・検証してから屋外で GPS に差し替えられる。
   **注意**: 2026-07-17 ログの `mocap_yaw_deg` は機体ヨーと無相関(~−1800° 回転、剛体マーカーの
   ヨー曖昧性)で **ヨー真値には使えない**。位置は使えるがヨー真値評価にはマーカー非対称化が要る。
3. **StampFly のホバは v ≈ 0.05〜0.1 m/s と静止的**なので、ホバ中は機構 A の稼ぎが小さく、
   **磁気+ジャイロコースト依存**になる。ホバ多用機の磁力レス化は常時前進する飛行機より難しい。
4. **民生 GPS の速度精度(~0.05〜0.1 m/s @ 5〜10Hz)は機構 B(微分で加速度)にはノイジー**。
   低高度・低速の StampFly ではフロー(機構 A)の方が優秀で、これも「フローを積む」判断を後押し。
5. GPS の要件(引用元より): UBlox M8 以上、飛行中の上空視界確保。旧世代 GPS では GSF は悪化する。

## 5. 推奨ロードマップ

前回検討(`OPTICAL_FLOW_STUDY.md` の案2/案3)を、位置のみ汎用化方針に沿って更新:

1. **オフラインリプレイ基盤(`data_analysis/replay/`、実装済み)上で拡張水平-ヨー EKF
   (ψ, b_gz, vN, vE)を試作**。MoCap 位置を真値・観測源として、飛行前にオフラインで挙動を詰める。
   実機試行錯誤をデスクトップに前倒しする勝負所。Code Identity で機上と同一挙動を保証(実測
   RMS 0.91mm で検証済み)。
2. **フロー観測(機構 A)を追加**して並進中のヨー可観測性を確認。PMW3901 ハードCS burst
   プローブ(実装済み)が drone3 で通れば 100Hz フローが使える。
3. **磁気は相補ソースとして残す**(ホバ担当)。GSF は当面 ride-along のクロスチェックとしてのみ、
   磁力レス化を決断した時にフル実装。
4. **GPS 統合は屋外展開時**。位置源の差し替えだけで、推定器本体は 1〜3 で完成させたものを流用。

**設計上の一線**: 実装は「4 状態に位置を足す」ではなく「速度を状態に入れた水平-ヨー統合
フィルタ」である。この一点を外さないこと。

## 6. 参考(可観測性の要点の一行まとめ)

- 位置はヨーを直接観測しない(∂z_pos/∂ψ = 0)。
- ヨー感度 = 機構 A では ‖v‖、機構 B では ‖a_水平‖。どちらも励振が無いとゼロ。
