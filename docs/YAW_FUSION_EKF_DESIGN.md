# 統合ヨーEKF(HY-EKF)設計書 — 生値からヨー推定までのフルロジック

作成日: 2026-07-18(同日、コード整合・数式・実現可能性の 3 視点による敵対的レビューを実施し反映済み)
位置づけ: **設計文書**(現行 V2 の実装仕様ではない。実装仕様の正典は PROTOCOL.md / ARCHITECTURE.md)。
`YAW_ESTIMATION_POSITION_ONLY_STUDY.md`(理論編: 可観測性の数式)を前提に、
**BMM150 生値・PMW3901 生カウント・MoCap 生位置・IMU 角速度/加速度をどう処理し、
どの不確かさモデルで重み付けし、最終的なヨー角推定に到達するか**を、現行 4 状態 EKF
(電流 FF の不確かさ σ_ff/σ_slew/σ_diff を積んで適応 R を組む粒度)と同じ granularity で定義する。

出典: 現行実装の仕様抽出(2026-07-18、`yaw_estimation/` 実コード)、フロー較正実測
(StampFly_Telemetry、drone3)、stampfly_ecosystem ESKF のフロー観測処理、
`FLIGHT_ANALYSIS_20260717.md` の実飛行データ。

## 0. 結論(最初に読む)

**現行 EKF より高精度なヨー推定は構築可能である。ただし無条件ではない。** 成立条件と
改善幅を最初に正直に述べる:

**改善が明確に見込める領域**
- **並進運動中(‖v‖ ≥ 0.3 m/s)**: フロー(機体系速度)× 世界速度(MoCap 位置由来)のペアが
  磁気と完全独立にヨーを拘束する。2 秒窓で **σ_ψ ≈ 2.5〜4°**(保守見積り、§7。系統誤差の床
  ~1.5〜2° を含む)。磁気が最も汚染される高スロットル・並進過渡で最も強い。
- **縮退暴走経路の排除**: 現行の「ホバ中に磁気残差が b_m に単調吸収 → ‖b_m‖>20µT 凍結 →
  ヨー角制御停止」という失敗経路(実飛行で |b_m| が凍結閾値の 93% まで成長: FLIGHT_ANALYSIS
  §2.8.2)を、可観測性ゲートと弱アンカー化(§5.2)で排除する。

**改善が条件付き・限定的な領域**
- **ホバリング中**: 新規ヨー情報源は存在しない(理論的限界)。改善は (a) b_m 暴走の遮断、
  (b) 直前の運動窓で得た ψ アンカーからの高品質コースト(b_gz 残差 0.04〜0.1°/s 級 → 4〜10°/90s)、
  (c) 磁気の弱アンカー化による FF 残差追従の抑制、の組合せで得る。**「1 桁改善」は主張しない**
  (レビューで棄却)。目標は「縮退暴走の根絶+ホバドリフト 2〜5 倍改善」であり、達成の実証は
  リプレイ A/B 比較(§9)が前提。
- **ホバ回頭(その場ヨー回転)**: ‖v‖≈0 のため機構 A/B とも感度ゼロで、磁気 EMA 遅れ
  (τ≈0.46s)が唯一のヨー情報のまま。**EMA 遅れ補償(§3.3 新設)が無ければ現行と同じ系統誤差**
  (0.5 rad/s 回頭で ~14° の引きずり)が残る。補償込みで現行より改善。

**ハード前提(いずれか欠けると機構 A が成立しない)**
1. **PMW3901 burst 読みの実機成立**(§3.4): 単発読み 6ms は 400Hz tick(2.5ms)と物理的に
   非両立。burst(実装済みプローブ・実機未検証)は「高速化オプション」ではなく**成立条件**。
2. **v_target のプロトコル追加**(§3.5): 現行の静的目標前提では、最もヨー情報が濃い
   軌道移動中に世界速度観測を自ら捨てる。
3. **運動窓が運用に存在すること**: 2026-07-17 実飛行(95s 定点ホバ、漂い速度 ~0.06 m/s)は
   運動窓ゼロであり、**現状の運用そのものでは機構 A は一度も作動しない**。数十秒毎の並進
   (ステップ目標移動 or 軌道モード)を運用に組み込むことが前提。

該当しない運用(完全静止ホバのみ・フロー不能床面)が支配的なら、本設計の投資対効果は低い
(§8)。その場合の正直な答えは「現行 EKF の FF モデル改良(電圧/duty 依存項)の方が先」である。

粒度の要約: 9 状態 EKF `x = [ψ, b_gz, b_mx, b_my, vN, vE, b_aX, b_aY, ψ_off]`。
現行 4 状態の磁気パイプライン(FF 補正+適応 R+NIS 二段ゲート)を観測の一つとして継承し
(数式・定数は不変。HY-EKF 側でゲート 2 点のみ変更: §3.3)、水平速度・実効加速度バイアス・
磁気フレームオフセットを追加。フロー観測・世界速度観測を新設し、項別の不確かさモデルを与える。

※ ベースライン数値について: 本文の「+20°/90s ドリフト」は 2026-07-17 ログ 2 本
(20260717_195243/195648)の explog 解析(セッション内リプレイ、est_mode=0 時のテレメトリ EKF)
による実測であり、リポジトリ恒久文書には |b_m| 18.6µT/90s 単調成長(FLIGHT_ANALYSIS §2.8.2)
として記録されている。また改善の実測評価は **MoCap 剛体マーカー非対称化(§10-5)が完了する
まで手段が存在しない**(現行 mocap ヨーは ±90° フリップ 2.5〜3.1%/行で真値に使えない)。

## 1. 設計原則

1. **現行資産の継承と変更の明示**: BMM150→較正→電流FF→EMA→レベル化→適応R→NISゲートの
   磁気パイプラインは実飛行で鍛えられた資産であり、**数式・定数を変更せず観測の一つとして
   使う**。HY-EKF 側で変更するのは **(a) b_m 更新の可観測性ゲート、(b) Z_REJECT の R 膨張
   降格、の 2 点のみ**(§3.3/§5.2)。現行 4 状態 EKF は無改変で並走(フォールバック兼比較対象。
   V2 契約 §2.1 と整合)。
2. **位置源の抽象化**: 世界速度観測のソースは「MoCap 位置微分」「GPS ドップラー速度」を
   同一インターフェース(v^W 観測+σ+鮮度)で差し替え可能にする。MoCap のヨーは使わない
   (方針)。MoCap の**位置**は GPS 位置の高精度代替として使う。
3. **不確かさは項別に積む**: 各観測の R は物理成分(量子化・ジャイロ漏れ・距離誤差・
   スケール誤差・遅延・受信ジッタ・有色性)の二乗和で組み、実測値で係数を決める。
4. **可観測性を明示的に管理する**: ヨー情報の流入量を Fisher 情報アキュムレータで定量化し、
   縮退時の状態更新をゲートする。**ゲートは「残差の行き先を付け替える」のではなく
   「引き込み速度を落とす」方向で設計する**(§5.2 — レビューで初版の b_m 単独凍結案は
   「残差が ψ に直撃する」として棄却)。

## 2. 状態ベクトルとモデル

### 2.1 状態(9 状態)

```
x = [ ψ,  b_gz,  b_mx,  b_my,  vN,  vE,  b_aX,  b_aY,  ψ_off ]
```

| 状態 | 意味 | 単位 | ダイナミクス |
|---|---|---|---|
| ψ | ヨー角(制御ワールド系) | rad | チルト運動学(現行と同一式) |
| b_gz | ヨージャイロバイアス | rad/s | RW(現行 q_bg=3.046e-12 継承) |
| b_mx, b_my | レベル化磁気バイアス | µT | GM τ=120s、q_bm=0.02 µT²/s(現行継承) |
| vN, vE | 世界水平速度(制御ワールド系) | m/s | v̇ = R_z(ψ)·(f_lev − b_a)(§2.2) |
| b_aX, b_aY | 実効水平加速度バイアス(レベル化機体系) | m/s² | GM τ=60s |
| ψ_off | 磁気アンカー系↔制御ワールド系の定数オフセット | rad | RW(q 極小) |

**b_a の設計意図と限界**: 加速度計バイアス(6 面較正が未適用)と Madgwick チルトバイアスの
**低周波成分**を吸収する実効状態。ただし **Madgwick チルト誤差は加速度と相関する過渡**
(水平加速中に姿勢が atan(a_h/g) 方向へ引かれ、位置制御のチルト指令と同期して秒スケールで
変動)であり、τ_ba=60s の GM 状態は追従できない。この過渡は Q_v の加速度相関項(§2.2)で
吸収し、**機構 B の情報量は保守的に扱う**(§7。機構 A 主体で設計する)。

**ψ_off の設計意図**: 磁気観測のアンカー(ψ0/B0、離陸方位基準)と制御ワールド系(MoCap 軸)
の定数差を**共分散付きで連続推定**する。初版のワンショット再アンカー案は「A/B 系が誤収束して
いた場合に飛行中唯一の独立絶対参照(磁気)を片道で破壊する」ためレビューで棄却。ψ_off は
b_m と同じくヨー励振下でのみ可観測なので §5.2 のゲートを共用する。簡易実装(v0)としては
ワンショット整合+安全ゲート(|Δψ0|<15°、実行時 b_m←0・P 再設定)を許すが、v1 は状態化を推奨。

**位置状態を持たない理由と代償**: MoCap 位置は微分+LPF で σ≈0.10 m/s の世界速度になり、
GPS はドップラー速度を直接与えるため、速度観測が本線(8+1 状態で足りる)。**代償**として
微分(MA(1) 負相関)+LPF(正相関)により観測ノイズが有色になり、白色扱いすると情報を約 2 倍
過大計上する — §3.5 の白色化措置(R×2 または 25Hz 間引き)で対処する。微分ノイズが性能律速と
判明した場合、位置 2 状態の追加(11 状態)を v2 とする。

### 2.2 予測ステップ(400 Hz、毎 tick)

```
ψ̇   = ((ω_z − b_gz)·cosθ − p·sinθ) / cosφ       ← 現行実装と同一式(検証済み・変更禁止)。
                                                     cosφ < 0.5 でフォールバック ψ̇ = ω_z − b_gz、
                                                     ±720°/s クランプも現行同一。
ḃ_gz = 0(+RW)
ḃ_m  = −b_m/τ_bm(+RW)                            τ_bm=120s
v̇^W  = R_z(ψ) · (f_lev − b_a)                     ← ψ が速度予測に入る = 機構 B を内包
ḃ_a  = −b_a/τ_ba(+RW)                            τ_ba=60s
ψ̇_off = 0(+RW 極小)
```

ヤコビアン F の非ゼロブロック(疎):

```
∂ψ̇/∂b_gz  = −cosθ/cosφ
∂v̇/∂ψ     = S·R_z(ψ)·(f_lev − b_a)               S=[[0,−1],[1,0]]
∂v̇/∂b_a   = −R_z(ψ)
```

プロセスノイズ Q(離散、×dt):

```
q_ψ  = 1.523e-7 rad²/s、 q_bg = 3.046e-12(現行継承)
q_bm = 0.02 µT²/s + GM(現行継承)
q_v  = σ_a_eff²
σ_a_eff² = 0.3² + k_d·Var(duty) + k_tilt·‖f_lev‖²   [m²/s⁴]
  第 2 項: 振動プロキシ(duty 4 本は 400Hz で使える最良の指標)
  第 3 項: 加速度相関のチルト過渡(§2.1)を Q 側で吸収 — 加速中は速度予測の確度を
           自動で落とし、機構 B が汚れた比力を過信しない(k_d, k_tilt はリプレイ同定)
q_ba = 2·σ_ba,ss²/τ_ba、 σ_ba,ss = 0.15 m/s²
q_ψoff ≈ (0.1°)²/60s(準定数)
```

初期 P0: 現行 4 状態の P0=diag(0.0305, 7.62e-5, 16, 16) を継承。v は (0.1 m/s)²、
b_a は (0.3 m/s²)²、ψ_off は (10°)² で開始。

### 2.3 実行位置と負荷

- 予測: 400 Hz tick 内、現行 EKF と同じ場所(sensorHub 更新)。
- 磁気更新: 現行どおり sensorHub 内 ~10 Hz(DRDY fresh 時)。
- 世界速度更新: `apply_pos_err` 直後の新フック(受信同 tick で消費)。
- フロー更新: **移植後に新設する**フロー読みスロット直後(sensor_read 内。ToF・磁気の既存
  20Hz スロットと位相分離)。V2 現状に PMW3901 統合は存在しない(Telemetry から移植)。
- 演算量: 9 状態 P 伝播 ≈ 2×9³ ≈ 1.5k 積和/step → 400 Hz で 0.6M MAC/s、ESP32-S3 で
  tick 予算 2500µs に対し ~5µs 級。観測更新は 2×2/スカラー逐次(Joseph 形)で ≪100µs。
- **稼働条件の再設計**: 現行 EKF は FF 無効時 predict すら走らない。HY-EKF は
  **センサ初期化後は常時 predict**。磁気観測のみ FF パイプライン健全時に限定し、
  フロー/世界速度観測は FF と独立に受け付ける。

## 3. 信号別パイプライン(生値 → 観測)

### 3.1 IMU ジャイロ(予測入力)

```
BMI270 生値(±2000dps→rad/s, 400Hz)
 → 軸変換(Roll_rate_raw=gyro_y, Pitch_rate_raw=gyro_x, ω_z=−gyro_z: 既存規約)
 → 起動時オフセット減算(800 サンプル 2s 平均: 既存)
 → [HY-EKF predict へ: 未フィルタのまま(現行 EKF predict と同規約。制御用 3ms LPF は通さない)]
```

- b_gz は状態で吸収。X/Y 軸バイアスは φθ 経由の二次項でしか効かないため状態に持たない。
- フロー観測のジャイロ補償(§3.4)には**フロー積分区間の平均ジャイロ**(400Hz リング
  バッファからフレーム間平均)を使う。
- **ψ 履歴リングバッファ**(400Hz×256 段 ≈ 640ms)を predict で常時更新する。磁気 EMA 遅れ
  補償(§3.3)と将来の遅延厳密化(§3.5)が共用する。

### 3.2 IMU 加速度(予測入力 — 新規使用)

```
BMI270 生値 [g](±8g, 400Hz)
 → 軸変換(Accel_x_raw=acc_y 等: 既存規約)
 → ×9.80665 で m/s² 化(★必須。高度KFの 9.81 倍バグの教訓 — 単位変換をパイプライン先頭で明示)
 → (推奨)NVS 済み 6 面較正の適用(現状未適用。適用しない場合は b_a が吸収するが収束が遅い)
 → レベル化: f_lev = Π_h[ R_x(φ)R_y(θ)·f^B ] の水平 2 成分
    ※回転規約は levelMagVector と同系(実機検証済みの非教科書符号)を踏襲し磁気と同一関数系を
      使う。符号の実機検証は §10-7(ここのミスは機構 B を正負逆に汚す)
 → 1 次 LPF(fc ≈ 15 Hz)
 → [HY-EKF predict の f_lev へ]
```

- 重力はレベル化で除去。チルト誤差起因の見かけ加速度: 低周波は b_a、加速度相関過渡は
  q_v の k_tilt 項が受け持つ(§2.1/§2.2)。
- **地上 ZUPT**: WAIT(接地)中 v=0 擬似観測(R=(0.02 m/s)²)で b_a を事前収束。
  ★効果は**加速度計バイアス分のみ**。飛行中の Madgwick 動的チルトバイアスは地上では
  現れないため、「離陸直後から機構 B がフル精度」の根拠にはならない(レビュー指摘)。

### 3.3 BMM150 磁気(観測 1 — 現行パイプライン継承+ゲート 2 点変更+遅れ補償)

```
BMM150 生値(ODR 10Hz, I2C, 20Hz スロットで DRDY ポーリング → 実効 10Hz)
 → Bosch トリム補償 [µT] → 軸変換(bx=+sy, by=+sx, bz=−sz)
 → 3D 楕円体較正: b_cal = M·(raw − offset)
 → 電流 FF 補正: b_corr = b_cal − ΔB̂(I)
    ΔB̂ = LUT(I_total) 区分線形 + Σã_m·δI_m(duty→電流モデル Î_m=c2d²+c1d+c0)
    INA3221 CH2 20Hz(AVG16)。同 tick 内で電流→磁気の順(既存契約)
 → EMA(α=0.18 @10Hz、τ_EMA = 0.1·(1−α)/α ≈ 0.46s)
 → レベル化(levelMagVectorBody: 非教科書符号・変更禁止)→ 水平 2 成分
 → 観測: h = R_z(ψ + ψ_off − ψ0)·B0_horiz + [b_mx, b_my]
    ★遅れ補償(新設): イノベーション評価には ψ の現在値でなく履歴値 ψ(t − τ_EMA)
      (§3.1 のリングバッファから取得)を使う。補償不能時のフォールバックとして
      R += (ψ̇_filt·τ_EMA·‖B0_horiz‖)² を加算。
      根拠: ホバ回頭 0.5 rad/s で遅れ残差 ≈ 7.5µT、NIS≈10 は棄却帯(13.8)に届かず
      「膨張採用」され ψ を系統的に逆方向へ引きずる(現行実測: 回頭ランプ中最大 21.9° 誤差)。
      ホバ回頭では機構 A/B の感度がゼロのため、この補償が唯一の対策(レビュー指摘)。
 → 適応 R = 4.0 + σ_ff² + σ_slew² + σ_diff² + (sin(tilt)·3.5)² [µT²]
    σ_ff = 0.03·|ΔB̂_xy|
    σ_slew = ‖a_xy‖·|dI/dt|·0.05s(a = LUT 局所傾き [µT/A]。方式 B の差動係数 ã_m とは別物)
    σ_diff = 9.0·max|δI_m|
    ノルム偏差 8–20µT で ×(dev/8)² 膨張(以上、現行そのまま)
 → ゲート群(現行を全て継承 — 移植時の欠落防止のため列挙):
    TILT_SKIP(tilt>25° はスキップ)/ ノルム偏差 >20µT ハード棄却 /
    NIS 二段(>5.99 で R×(NIS/5.99) 膨張採用、>13.8 棄却)+
    ソフト再捕捉(time_since_accept>5s なら >13.8 でも制限付き更新: R 膨張+バイアス行
    K ゼロ化+Δψ≤3°/更新)/ 連続棄却 3s で P 緩膨張 1.02/s(≤10×P0)
```

**HY-EKF 側で変更する 2 点**(現行 4 状態 EKF は無改変):
1. **b_m/ψ_off 更新の可観測性ゲート**(§5.2)。
2. **Z_REJECT の降格**: 現行の「機体 z 残差 >12µT で全棄却」は飛行中 91–95% 常時成立で
   判別力ゼロ(実証済み)のため、HY-EKF ではハード棄却をやめ R 膨張に変更:
   `R_eff ×= 1 + (z_res / 12µT)²`(無次元倍率。κ の初期値 1、リプレイで同定)。

### 3.4 PMW3901 フロー(観測 2 — 新設。機体系速度)

**★ハード前提**: 単発読みは 1 サイクル ≈6.0–6.5ms(t_SRAD=500µs×9 トランザクション)で、
400Hz tick 予算 2.5ms に**物理的に入らない**(2–3 tick を潰し、同一 SPI バスの BMI270 毎 tick
読みもブロックする)。したがって **burst 読み(ハードCS、13B@2MHz≈52µs、eco で 100Hz 飛行実績、
V2 用プローブ実装済み・実機未検証)の成立が機構 A の成立条件**である。burst 不成立の場合は
(a) 機構 A を断念(NO_FLOW 恒常)か、(b) フロー読みを別コアタスク化+BMI270 とのバス調停を
新設計する(工数大)の二択(§10-2)。

```
生カウント Δx_raw, Δy_raw + SQUAL + Shutter(burst 13B 一括、50–100Hz)
 → dt_f: 読みスロット micros 実測差(現行 Telemetry 実装。初回のみ公称 20ms)。
   HY-EKF 移植時に健全化ガードを新設: elapsed∉(0, 1s) は公称値(50Hz=0.02s/100Hz=0.01s)へ再シード
 → 搭載向き remap(Telemetry 検証済みマッピング)→ カウントレート raw/dt_f
 → スケール: flow = raw_rate / k_scale
    k_scale(drone3 実測・較正適用済み): x=472.3 counts/rad (r²=0.88) / y=471.8 (r²=0.88)
    ※機体毎に flowcal 必須(合格: n≥200, r²≥0.80, 励振 std≥0.30 rad/s, x/y 比≤1.2)
 → ジャイロ補償(フレーム間平均ジャイロ): trans_x = flow_x − q̄, trans_y = flow_y + p̄
 → 距離乗算: v^B = [trans_x·d, trans_y·d]
    d = ToF 生スラント距離 Range[mm]/1000(検証済み: cosφcosθ は掛けない。有効 0.08–2.0m)
    ToF 無効フレームは前回値ホールド(>200ms 続けば観測停止)
 → 観測: h = R_z(ψ)ᵀ·[vN, vE]
    H: ∂h/∂ψ = R_z(ψ)ᵀ·Sᵀ·v^W(ノルム=‖v‖: 機構 A の感度)、∂h/∂v = R_z(ψ)ᵀ
```

**R_flow(項別、軸ごと、毎フレーム)**:

```
σ_v² = d²·(σ_rate² + σ_gyro²)      … 量子化+SQUAL 依存+ジャイロ漏れ
     + trans²·σ_d²                  … 距離誤差(乗算的)
     + (v̂_axis·σ_k/k)²             … スケール誤差(回間ばらつき実測 2.5%。セッション内は
                                       系統誤差 — §7 では 1/√N 圧縮しない側に分類)
     + σ_base²                      … 床面テクスチャ等の床(0.05 m/s)

σ_rate = σ_count(SQUAL) / (k_scale·dt_f)
  σ_count(SQUAL): ≥80→0.7 / 60–80→1.0 / 40–60→2.0 / 25–40→4.0 counts、<25 棄却
  (SQUAL→σ 対応は両既存実装に無く新設。白机実測 平均 79–88/p5≈55 を基準に設計、
   PX4 の R 補間 0.15–0.5 m/s を包絡としてリプレイで確定)
σ_gyro ≈ 0.05 rad/s、σ_d = 0.02 m
```

数値例(d=0.6m, SQUAL 80, v=0.3 m/s): σ_v ≈ 0.075 m/s(eco の固定 R=0.30 m/s は保守包絡)。

**ゲート**: SQUAL≥0x19、Shutter_Upper≠0x1F、|レート|≤7.4 rad/s、d∈[0.08, 2.0]m、
イノベーションクランプ ±0.3 m/s(eco 方式: 飽和)、NIS χ²(2)>9.21 棄却。
取付オフセットの ω×r 項は ≤0.01 m/s で σ_base に含む。

### 3.5 MoCap 位置(観測 3 — 新設。世界系速度。GPS に差し替え可能)

**★プロトコル前提**: 現行 CMD_POS_ERR は静的目標しか区別できず(目標速度フィールド無し)、
軌道モード(円/8 の字)中は d(err)/dt = v_target − v^W となって観測が成立しない。
**軌道中こそ ‖v‖>0 でヨー情報が最も濃い**ため、v_target(2×int16 [mm/s]、21→25B)の追加は
「推奨」ではなく**機構 A の実質必須要件**とする(bit4=target_static のみの暫定案は、
ステップ目標移動時しか運動窓を作れない縮退構成としてのみ許容)。

```
OptiTrack 生位置(PC 側: 既存の外れ値/ジャンプフィルタ済み)
 → CMD_POS_ERR @50Hz: err = target − pos(制御ワールド系, ±2.0m クランプ)+ v_target(追加)
 → 機上: v^W_meas = v_target − Δerr/Δt_nom
    Δt_nom = 20ms × Δseq(受信時刻差でなく送信規約の公称値を使う — ESP-NOW 受信ジッタ
    ±3–5ms が ‖v‖ 比例の乗算誤差(v=0.4 で ~0.1 m/s)になるのを避ける。レビュー指摘)
 → 1 次 LPF fc≈8Hz
 → 観測: h = [vN, vE](H は v ブロック単位行列)
```

**R_vel(項別)**:

```
σ_vel² = ( 2·σ_p²/Δt_nom² · G_lpf )·κ_color        … 微分増幅(LPF 利得補正)×有色性補正
       + (‖f_lev − b_a‖·τ_d)²                       … 遅延×加速度
       + (‖v̂‖·σ_Δt/Δt_nom)²                        … 残留タイミング誤差(σ_Δt≈2ms)
τ_d = mocap 5–10ms + ESP-NOW 2–10ms + 微分中心 10ms + LPF 群遅延 1/(2πfc)≈20ms ≈ 40–60ms
    (初版の 25–40ms は LPF 群遅延の計上漏れ — レビュー指摘で改訂)
κ_color = 2(微分の MA(1) 負相関+LPF 正相関を白色扱いする補正。リプレイで NIS 平均≈自由度に
    なるよう同定。代替: 更新を 25Hz に間引く)
σ_p ≈ 2mm → σ_vel ≈ 0.10–0.15 m/s(加速時はさらに増)
```

**ゲート**(レビューで実在イベントへの防御 3 点を追加):

```
bit2(XY 有効)∧(v_target 有効 or bit4)∧ age<100ms ∧ seq 単調増加 ∧ Δseq≤2
+ err 停滞検出: err が連続 N=4 回同値なら MoCap 無音途絶とみなし観測停止
  (PC は無音途絶を 300ms まで bit2=1 のまま送る実装 — 凍結 err の微分=0 が
   「偽の静止観測」として素通りするのを防ぐ)
+ NIS χ²(2) > 9.21 棄却 + イノベーションクランプ ±0.5 m/s
  (復帰時の err ジャンプ(実測 253mm → 微分 12.5 m/s スパイク)対策)
+ bit2 の 0→1 復帰後 0.4s は微分履歴を再シードし観測を捨てる(XY PID の D 再シード規約と同型)
```

MoCap 途絶時は本観測を止め、既存フェイルセーフ階層(機上 >200ms 水平保持 / >500ms 自動着陸)
には介入しない。**GPS 差し替え**: 同一インターフェースに GPS ドップラー速度(σ≈0.05–0.1 m/s、
5–10Hz、τ_d≈100–200ms → 遅延項支配、必要なら §3.1 の ψ 履歴で固定ラグ化)を供給。

### 3.6 使わない生値と理由

- **MoCap ヨー**: 方針により不使用(評価の参照信号としてのみ。マーカー非対称化が前提)。
- **気圧**: 高度系(別フィルタ)の担当。
- **加速度からの Roll/Pitch 再推定**: Madgwick の役割(責務分離。チルト誤差は b_a+q_v が吸収)。

## 5. ゲート・健全性・縮退

### 5.1 縮退ラダー

```
FULL      : 磁気 + フロー + 世界速度      … 通常
NO_FLOW   : 磁気 + 世界速度              … フロー品質低下(機構 B のみ=加速時だけヨー情報)
NO_EXT    : 磁気のみ                     … MoCap/GPS 途絶(≒現行 EKF 相当)
COAST     : ジャイロ+b_gz のみ           … 磁気も不健全
```

ソース健全性: フロー = vel_valid ∧ fhr>0.5 / 世界速度 = §3.5 ゲート / 磁気 = 現行 FF 健全判定。
モード遷移は状態連続・共分散膨張のみ(現行の「フレーム差ラッチ」パターン踏襲)。

### 5.2 ψ/b_m/ψ_off 縮退の可観測性ゲート(設計改訂版)

**Fisher 情報アキュムレータ**(10s 窓、単位 [rad⁻²]):

```
J_ψ = Σ H_ψᵀ R⁻¹ H_ψ
    = Σ_flow+vel同時健全 ‖v̂‖²/σ_pair²  +  Σ_vel健全 (‖f_lev‖·Δt_upd)²/σ_vel²
      (機構 A 項)                          (機構 B 項。Δt_upd=更新間隔 20ms。
                                            ★初版は Δt² 因子を欠き情報を ~2500 倍過大計上
                                            → ホバ振動だけでゲートが開く欠陥。レビューで修正)
E_ψ = 窓内 |Δψ| 経路長
```

**ゲート動作**(初版の「b_m 行 K ゼロ化」単独案は、ホバ中の磁気残差の接線成分が
**全量 ψ に直撃する**(B0h≈30µT に対し実測 18.6µT 級の残差 → 最悪 ~35° の ψ 偏差ポテンシャル)
としてレビューで棄却。改訂版は「引き込み速度の制限」方式):

```
J_ψ > J_min ∨ E_ψ > 20° のとき(可観測):
  磁気更新はフル動作(b_m・ψ_off 行も更新)
それ以外(縮退中 = ホバ):
  (1) b_m・ψ_off 行の K をゼロ化(+b_gz 行も同時ゼロ化 — 現行ソフト再捕捉が
      b_g 行を同時に落とすのと同じ理由: 大残差の b_gz 汚染防止)
  (2) 磁気の実効 R に下限膨張 κ_hover(≥2)を掛け、ψ への引き込み時定数を
      τ_pull ≈ 30–60s に制限(=「弱アンカー化」。FF 残差の成長に追従せず、
      かつコーストの無限ドリフトは防ぐ、の釣り合い点。κ_hover はリプレイで
      b_gz コースト精度と釣り合うよう同定)
  (3) イノベーション持続バイアス検出は**常時稼働**(運動窓限定にしない —
      初版の欠陥): |mean(innov)|>3σ(20s 窓)で R×4 を 10s 保持
```

実装は「状態・共分散で同一の制限ゲインを使う K 行ゼロ化」パターンを新規実装で踏襲する
(現行ソフト再捕捉はインライン処理であり、そのままの流用はできない)。

**このゲートで断てるもの/断てないもの**(正直に): 断てるのは「b_m 単調成長 → 20µT 凍結 →
ヨー角制御停止」の暴走経路と、FF 残差への高速追従。断てないのは「ホバ中の磁気起因ドリフト
そのもの」— 弱アンカーの τ_pull と b_gz コースト品質で決まる残留ドリフトは残る。
ゲート ON/OFF × 実飛行ログ(18.6µT 成長便)のリプレイ比較で副作用が無いことを示すのが
§9 の必須検証項目。

### 5.3 磁気クロスチェック

運動窓中: A/B 由来 ψ と磁気の持続乖離で FF モデル不整合を検出(→ R×4)。
ホバ中: §5.2-(3) の持続バイアス検出が常時代替。GSF の ride-along 思想の StampFly 版。

## 7. 期待精度の見積り(レビュー反映後の保守計算)

**前提条件を最初に明示**: 以下は「運動窓(‖v‖≥0.3 m/s、2s 以上)が運用に組み込まれている」
場合の見積りである。2026-07-17 実飛行のような定点ホバのみの運用では運動窓はゼロであり
(漂い速度 ~0.06 m/s)、機構 A は作動しない(§8-1)。

- **観測の実効 σ**: σ_flow ≈ 0.075 m/s、σ_vel ≈ 0.10–0.15 m/s →
  σ_pair ≈ 0.13–0.17 m/s(白色成分)+系統成分(スケール 2.5%・遅延×加速度バイアス・
  フレーム整合残差)≈ 相当 1.5–2°。
- **運動窓(v=0.4 m/s、2s、50Hz)**: σ_ψ² = σ_white²/N_eff + σ_sys²(N_eff≈50: 有色性で半減)
  → **σ_ψ ≈ 2.5〜4°/窓**(系統床込み)。
- **b_gz**: 運動窓 1 回の情報(~1.7°/s)は事前分布(起動較正後 0.3–0.5°/s)より 1 桁弱く、
  **窓単発では実質改善しない**(初版の「窓 3 回で 0.01–0.02°/s」は導出不能な誤り — レビューで
  棄却)。b_gz の実効的な拘束は**窓間の ψ アンカー差 / 経過時間**で決まり:
  σ_bg ≈ √2·σ_ψ_window/ΔT_sep → ΔT_sep=30–90s で **0.04–0.12°/s**。
- **ホバコースト(運動窓後)**: 4〜10°/90s(b_gz 残差起因)+磁気弱アンカーによる頭打ち。
  現行実測(+20°/90s 級、est_mode=0 時のテレメトリ EKF 解析)に対し **2〜5 倍の改善**が
  実証目標(1 桁は主張しない)。かつ「b_m 凍結→制御停止」の暴走経路は構造的に消える。
- **ホバ回頭**: EMA 遅れ補償(§3.3)により現行の系統誤差(回頭ランプ中最大 21.9° 実測)を
  遅れ補償誤差(ψ̇ 推定誤差×τ_EMA ≈ 1–2°)まで低減。※これは機構 A/B でなく遅れ補償の効果。
- **並進+磁気汚染時(高スロットル過渡)**: 機構 A が支配し、磁気 R の適応膨張と合わせて
  現行より明確に改善(現行はこの領域で ffg ゲート棄却率 ~90%)。

## 8. 現行 EKF より高精度にならないケース(正直な列挙)

1. **完全静止ホバのみの運用**: 新規ヨー情報ゼロ。改善は b_m 暴走遮断+弱アンカー化+
   (あれば)過去の運動窓由来アンカーに限られる。**2026-07-17 の実飛行(95s 定点ホバ、
   漂い 0.06 m/s)はまさにこのケースに該当する** — 現状の運用を変えずに HY-EKF だけ入れても
   機構 A は一度も作動しない。運用に並進を組み込めないなら、投資対効果は
   「FF モデルの電圧/duty 依存項の改良」の方が高い。
2. **burst 読みが実機不成立**(§3.4): 機構 A の成立条件が崩れ、NO_FLOW 恒常(機構 B のみ)。
   StampFly のホバ主体運用では機構 B の稼働窓(線形加速)が少なく、改善は限定的。
3. **v_target 追加を見送った場合**: 軌道モード中(最良のヨー情報源)に観測を自ら捨てる。
   ステップ目標移動のみが運動窓となり、改善機会が大幅に減る。
4. **フロー不能環境**(暗所・無テクスチャ床・SQUAL<25 恒常)。
5. **機体毎 flowcal 未実施**: x/y スケール不一致(合格条件は比 1.2 まで許容)で最大 ~5° の
   ヨーバイアス。drone3 は x/y 比 1.001 で問題なし。個体不良機(drone test)は不可。
6. **フレーム整合(ψ_off)の検証不足**: 磁気と A/B の綱引きがイノベーションバイアスとして
   残り、NIS ゲート誤作動の要因になる。pos_ctrl_yaw_sign の地上検証と同時に実施(§10-4)。
7. **1–2 Hz 超の高速ヨー振動**: 本フィルタの守備範囲外(レートループの領分)。
